Версия: 1.0  
Дата: 2026-02-05  
Статус: Production-ready specification

## 1. Дизайн-концепция

### 1.1 Tone of Voice интерфейса

**"Уверенный инструмент для людей, которые делают"**

Интерфейс не развлекает, не обучает, не ведёт за руку — он даёт контроль. Это dashboard для управления своими проектами, а не чат с ботом.

Ключевые характеристики:
- **Прямолинейность** — никаких "А вы уверены?" без причины
- **Информационная плотность** — много данных на экране, но структурированно
- **Немногословность** — короткие лейблы, без лишних пояснений
- **Профессиональный минимализм** — чистый, но не пустой

### 1.2 Пользовательский портрет

**Indie-хакер / вайб-кодер / микро-фаундер**

- Технически грамотен, не нуждается в объяснениях базовых вещей
- Ценит время — хочет сделать и уйти
- Привык к инструментам: Notion, Linear, Vercel Dashboard
- Ожидает отзывчивость и предсказуемость
- Возраст: 20–40, digital native

### 1.3 Чем отличается от Telegram-ботов

| Telegram-бот | VibeMom Mini App |
|--------------|------------------|
| Последовательный диалог | Произвольная навигация |
| Один экран = одно сообщение | Несколько зон информации |
| Reply/Inline кнопки | Полноценные формы, табы, карточки |
| Нет состояния на экране | Весь контекст виден сразу |
| "Бот спрашивает — ты отвечаешь" | "Ты управляешь — система реагирует" |

## 2. Навигация

### 2.1 Основной Layout

```
┌─────────────────────────────────┐
│  Header (контекст + действия)   │  40px
├─────────────────────────────────┤
│                                 │
│                                 │
│         Content Area            │  flexible
│                                 │
│                                 │
├─────────────────────────────────┤
│      Bottom Navigation          │  56px + safe area
└─────────────────────────────────┘
```

**Bottom Navigation** — основной способ перемещения между разделами. Выбран вместо side menu, потому что:
- Thumb-friendly (большие пальцы)
- Привычен для мобильных приложений
- Показывает все разделы сразу

### 2.2 Bottom Navigation Bar

5 пунктов (максимум для комфортного тапа):

| Иконка | Label | Экран |
|--------|-------|-------|
| ◻️ | Home | Dashboard |
| 📁 | Projects | My Projects |
| 🏪 | Catalog | Catalog |
| 📥 | Requests | Requests |
| 👤 | Profile | Profile |

**Leads** — вложен в Profile или доступен через Projects (у каждого проекта свои лиды).

### 2.3 Header

Контекстный header меняется в зависимости от экрана:

```
┌─────────────────────────────────┐
│ ← Back    Screen Title    [⋮]  │
└─────────────────────────────────┘
```

- **Back** — появляется на вложенных экранах (детали проекта, редактор)
- **Title** — название текущего экрана
- **Action** — контекстное действие (меню, фильтры, добавить)

### 2.4 Понимание "где я сейчас"

1. **Active state в Bottom Nav** — заливка/подсветка активной иконки
2. **Breadcrumb-паттерн в заголовке** — "Projects → Project Editor"
3. **Визуальная иерархия** — вложенные экраны имеют Back arrow
4. **Единый цвет акцента** — активные элементы одного цвета

### 2.5 Возврат назад

- **Hardware Back** (Android) — всегда работает, возвращает на предыдущий экран
- **Back Arrow в Header** — дублирует hardware back
- **Bottom Nav** — сбрасывает стек навигации раздела, возвращает на root

Правило: **Back всегда предсказуем**. Если пользователь нажал Back — он вернётся туда, откуда пришёл, без сюрпризов.

## 4. UI-паттерны

### 4.1 Карточки (Cards)

**Структура:**

```
┌─────────────────────────────────┐
│ [Status Badge]                  │
│                                 │
│ Title                           │
│ Subtitle / Description          │
│                                 │
│ Meta: price · niche · author    │
│                                 │
│ [Secondary Action] [Primary →]  │  (optional)
└─────────────────────────────────┘
```

**Правила:**
- Радиус скругления: 12px
- Padding: 16px
- Shadow: subtle (0 2px 8px rgba(0,0,0,0.08))
- Tap state: scale(0.98) + opacity(0.9)

### 4.2 Статусы (Status Badges)

| Статус | Цвет | Иконка | Label |
|--------|------|--------|-------|
| DRAFT | Gray (#6B7280) | ○ | Draft |

| APPROVED | Green (#10B981) | ✓ | Approved |
| REJECTED | Red (#EF4444) | ✕ | Rejected |
| ARCHIVED | Gray (#9CA3AF) | 📦 | Archived |

**Визуал:**
- Pill-shaped badge
- Background: цвет с 10% opacity
- Text + icon: цвет на 100%
- Font: 12px medium

### 5.5 Почему интерфейс "дышит"

1. **Достаточно белого пространства** — 16px margins, 12px gaps
2. **Иерархия через размер, не плотность** — крупные заголовки, компактные meta
3. **Группировка** — связанные элементы близко, разные — с отступом
4. **Ритм** — одинаковые отступы повторяются
5. **Один фокус на экране** — главное действие выделено, остальное приглушено

## 6. Принципы, которые нельзя нарушать

### 6.1 Preview == Published

Что пользователь видит в Preview — то же самое будет в канале. Один renderer, zero расхождений. Нельзя добавлять/убирать/переставлять элементы между preview и публикацией.

**Почему:** Доверие. Пользователь должен точно знать, что получит.

### 6.2 Никаких внезапных действий

- Любое деструктивное действие (удаление, архивация, отправка) требует подтверждения
- Confirm dialog должен чётко описывать последствия
- Кнопка подтверждения — справа, отмены — слева
- Деструктивные кнопки — красные

**Почему:** Пользователь не должен бояться нажимать кнопки.

### 6.3 Минимум кликов

- Dashboard → Create Project: 1 tap
- My Projects → Edit: 1 tap
- Project Editor → Submit: 1 tap + 1 confirm
- Catalog → Project Details: 1 tap

**Правило трёх кликов:** любое ключевое действие достижимо за ≤3 тапа.

### 6.4 Всё редактируемо

- Draft: всегда редактируемо
- Needs Fix: редактируемо
- Pending: только просмотр (ждём модерацию)
- Approved: можно архивировать
- Rejected: можно редактировать и переотправить

**Почему:** Пользователь должен чувствовать контроль над своими данными.

### 6.5 Всегда понятно, что будет дальше

- Кнопки говорят о действии: "Submit", не "OK"
- После действия — feedback: toast, redirect, state change
- При ошибке — понятное сообщение + как исправить
- Pending state — индикатор "Ожидает модерации"

**Почему:** Неопределённость = тревога. Определённость = доверие.

### 6.6 Mobile-first, Touch-friendly

- Минимальный tap target: 44×44px
- Отступ между интерактивными элементами: ≥8px
- Thumb zone: ключевые действия в нижней части экрана
- No hover-only states (всё доступно по tap)

### 6.7 Offline-tolerant

- При потере соединения — показываем последние данные + banner "Offline"
- Формы автосохраняются локально
- При восстановлении — синхронизация без потери данных
- Ошибки сети — retry с экспоненциальным backoff

## 7. Состояния и анимации

### 7.1 Loading States

**Skeleton:**
- Используем вместо spinner для списков/карточек
- Пульсирующая анимация (opacity 0.5 → 1.0)
- Форма соответствует контенту

**Inline Loading:**
- Spinner в кнопке при submit
- Disabled state на время загрузки

### 7.2 Transitions

- Page transitions: slide (300ms ease-out)
- Modal: fade + slide up (250ms)
- Toast: slide down (200ms)
- Card tap: scale(0.98) (100ms)

**Правило:** анимации быстрые (≤300ms), subtle, не блокирующие.

### 7.3 Haptic Feedback (если поддерживается)

- Tap: light impact
- Submit: medium impact
- Error: error pattern
- Success: success pattern
